Local Landing Page Playbook

Northwest Custom Apparel — how to spin up a high-ranking local page per city, the right way. For Mehar.

What this is & how to use it

The companion file nwca-local-landing-page-template.html is a complete, SEO-optimized example page for Tacoma. This playbook tells you how to turn it into a page for any service-area city — with the metadata, schema, and copy already structured — and, just as important, how to do it without creating thin duplicate pages that Google will ignore or penalize. The template is a reference build; recreate it in Elementor (or use it as an HTML base) — the structure, headings, schema, and internal links are what matter, not the exact styling.

⚠ Read this first — the #1 way local pages fail

Do not create pages that are identical except for the city name. Google treats find-and-replace "doorway pages" as spam and will either not rank them or suppress the whole set. The Bonney Lake page ranks because it's genuinely useful; scale that, not a mail-merge.

Every city page needs real, unique local content: different intro copy, the actual neighborhoods/towns nearby, real industries in that city, genuine testimonials from customers in that area, and specifics (drive time, local delivery notes, a local project example). Aim for at least ~40–50% unique body content per page. If you can't write something true and specific about serving that city, don't publish the page yet.

1. URL pattern & the cannibalization fix

Pick one URL pattern and use it for every city. Right now the site mixes /custom-embroidery-seattle/, /seattle-custom-t-shirts/, /spokane-dtg-and-embroidery/ and /embroidery-screenprinting-bonneylake-wa/. Standardize on:

/custom-embroidery-{{city-slug}}-wa/

301-redirect the old inconsistent URLs to the standardized ones. Critical: only one page should target "custom embroidery near me / [city]" per city — today the Seattle and Bonney Lake pages both chase "custom embroidery near me," which splits ranking strength and holds both on page 2. One city = one primary local page.

4. Structured data (schema)

Two JSON-LD blocks live in the template's <head>: a LocalBusiness + Service graph and a FAQPage. Rules:

  • Swap areaServed city, and the FAQ questions/answers, to match each page.
  • The FAQPage schema must match the visible FAQ text exactly — mismatched schema is a manual-action risk.
  • NAP in the schema must be byte-for-byte identical to Google Business Profile and the visible footer.
  • Validate every page in Google's Rich Results Test before publishing.
Use one canonical LocalBusiness @id (the main business) across all pages and vary only areaServed / Service. Don't invent a separate "business" per city — you have one business serving many areas.

6. Internal linking rules

  • Link every city page from a "Service Areas" section in the footer or main nav (so they're discoverable and get authority).
  • Each city page links out to the core service pages and 2–3 relevant blog posts.
  • Link nearby city pages to each other ("We also serve {{NEIGHBORS}}") — but only where genuinely relevant.
  • Add these pages to the XML sitemap and request indexing in Search Console after publishing.

7. Priority cities to build

Rebuild the existing under-performing local pages to this standard first (they already have keyword footprints), then expand:

  • Rebuild: Seattle, Spokane, Portland, Kent — currently earning ~0 traffic despite ranking for dozens of keywords.
  • Add (core service area): Tacoma, Milton (home city), Puyallup, Federal Way, Auburn, Lakewood, Fife, Kent, Renton.
  • Prioritize by local search volume for "custom embroidery [city]" / "screen printing [city]" and by where your real customers already are.

8. Pre-publish checklist (per page)

  • Unique intro + local specifics written (not just city-name swapped)
  • Title ≤ 60 chars, description ≤ 155, single H1, keyword-first
  • URL follows /custom-embroidery-{{city-slug}}-wa/; old variants 301'd
  • Only one page targets "custom embroidery near me / [city]" for this city
  • LocalBusiness + FAQPage schema updated; passes Rich Results Test
  • NAP matches Google Business Profile exactly (schema, body, footer)
  • Real reviews & testimonials only; ratings accurate
  • Internal links in (from Service Areas) and out (to service + blog pages)
  • Hero CTA + click-to-call above the fold; final quote CTA present
  • Image alt text includes city + service; images compressed
  • Mobile check; template notes banner removed
  • Added to XML sitemap; URL submitted in Search Console
