Read before you delete

  • "Not used" here = no reference in code (proxy/frontend/inksoft) and no Caspio wiring (views, relationships, tasks, webhooks). It does NOT check DataPages or Flex apps — the Caspio API can't reveal a DataPage's source table. Verify each candidate against the Caspio DataPages list + the "Christmas Bundles" Flex app before archiving.
  • Archive ≠ delete. Caspio blocks table DELETE via API by design. Safe path in the Caspio UI: export a CSV backup → rename to zz_ARCHIVE_<name> → wait a week → delete if nothing breaks. Start with the red "dup/old" tier.
  • The signal bias is safe: exact-name matching can only over-count usage, so a truly-used table is never mislabeled unused. Snapshot: 2026-07-10.
