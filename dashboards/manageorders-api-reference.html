Swagger vs reality — read before you build a push

  • Dates are MM/DD/YYYY on push (the Swagger says ISO — OnSite rejects ISO).
  • TaxTotal: 0 — OnSite drops the order-level tax on import. Put tax in a Notes block; the rep applies it via the ShopWorks dropdown. Tax flags sts_EnableTax01–04 (undocumented in Swagger) drive tax calc if used.
  • Color = CATALOG_COLOR (mainframe code), never COLOR_NAME. id_Design: omit (don't send 0) when unknown. Size: push the BASE part# + plain size — OnSite appends _2X/_3X.
  • Only 4 valid Notes[].Type values — one bad Type aborts the WHOLE push. ExtOrderID/ExtDesignID must be GLOBALLY unique (full date+seq+year).
  • GET /order-pull returns fields at the TOP level (Swagger shows them wrapped in order_json — it's wrong). Retention ≈ 2 years, not 60 days.
