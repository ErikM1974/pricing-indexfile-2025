The gotchas that actually break integrations

  • Color = catalogColor / mainframe code ("BrillOrng"), never the display COLOR_NAME — for every SanMar API, inventory, and order call. (One exception: our proxy's /api/inventory product feed keys on COLOR_NAME.)
  • Inventory caps at 3000/warehouse — a returned "3000" means ≥3000, not an exact count (both inventory services).
  • sizeIndex is a small ordinal (1=XS, 2=S, 3=M, 4=L, 5=XL, 6+=2XL…). uniqueKey = inventoryKey+sizeIndex = PromoStandards partId — the CSV merge key.
  • Use PRODUCTION WSDLs even for read testing — the Test env's products/pricing/inventory are stale. Test credentials are separate.
  • Order tracking: error 301 = order not shipped+invoiced yet (persisting >48h → flag + pause); 303 = date older than 7 days; allOpen excludes Complete — a fast order can vanish between polls (pair with invoice-based catch-up).
  • Standard getPricing = cost tiers, not MSRP, and price varies by color. MSRP/MAP = PromoStandards priceType=List or the mapPrice field (new Jun 2026).

Services & WSDLs prefix with https://<host>:8080
