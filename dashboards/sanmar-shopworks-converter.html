Convert a price list

Pick a SanMar price list and click Convert. It runs entirely in your browser (nothing is uploaded), then you download a CSV ready to import into ShopWorks → Parts. It auto-detects the file: a ShopWorks Integration Only Price List (already formatted) gets the sizes added to its descriptions and _2XL→_2X fixed; a raw SanMar feed is fully converted (colors collapsed, part numbers, prices, size flags, sizes in descriptions).

What the converter does

  • Descriptions get the size. Extended-size rows become Size 3XL - …, Size 2XL - …, Size OSFA - … — matching how your existing parts read.
  • 2XL part numbers are fixed. _2XL → _2X so an import updates your existing parts instead of creating duplicates. Ladies _XXL parts are left alone.
  • One part per style + size. On a raw feed, colors collapse to a single part (highest/standard piece price), with the ShopWorks size-limit flags set (Size01–04 base, Size05 for 2XL/XXL, Size06 for everything else).
  • Prices & brand carried through. Piece / Dozen / Case pricing and brand come straight from the feed.

Two ways to get a price list

Use ShopWorks's monthly "Sanmar Integration Only Price List", or pull the raw SanMar feed yourself from the SanMar Downloads page / FileZilla. Either one works here.

Then import into ShopWorks

  1. Download the CSV above.
  2. In ShopWorks (OnSite), import it into Parts — it matches on ID_Product, so existing parts update and new ones are added.
  3. Spot-check a couple of styles (e.g. 29M_2X reads Size 2XL - …).
